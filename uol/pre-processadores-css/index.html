O que é um pré-processador e o que ele faz?

Quais pré-processadores de CSS existem?


(mais utilizados)

Quais os principais problemas ao desenvolver CSS?



  • Falta de recursos básicos: variáveis e operadores matemáticos
  • Dificuldade na manutenção de grandes estruturas
  • Repetição de código



Quais as vantagens de se utilizar um pré-processador de CSS?

  • Variáveis
  • Operadores matemáticos
  • Reaproveitamento de código com Mixins
  • Criação de funções
  • Escrita de código hierarquica
  • Modularização do CSS em arquivos separados
  • Melhora o fluxo de trabalho em equipes de grande porte

Quais as desvantagens de se utilizar um pré-processador de CSS?

  • Aprender as variáveis/funções/recursos do pré-processador
  • Preparar ambiente de desenvolvimento (comum em qualquer desenvolvimento)
  • Em equipes de grande porte, manter o ambiente de desenvolvimento igual (versão ruby, versão pré-processador, etc)

Exemplos de código

Sass : o que é necessário para utilizar?

  • Por padrão: funcionamento server-side
  • Sistema: Windows, Mac OS, Linux
  • Instalar Ruby
  • Instalar o Sass
  • Editor de texto
  • Documentação oficial: http://sass-lang.com/
  • Fazer! xD

Less : o que é necessário para utilizar?


  • Por padrão: funcionamento client-side (processamento repassado ao usuário)
  • Opção: Funcionamento Server-side utilizando Node.js (compila o css)
  • Sistema: Windows, Mac OS, Linux
  • Utilizando versão standard: incluir js de interpretação na sua página
  • Utilizando versão compilada: Instalar Node.js
  • Editor de texto
  • Documentação oficial: http://lesscss.org/
  • Fazer! xD
